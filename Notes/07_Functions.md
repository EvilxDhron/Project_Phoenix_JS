# Traditional Functions and Modern Functions

## Q.1 What is a function?

A function is a reusable block of code designed to perform a specific task. It can accept inputs through parameters and return outputs using `return` statements.

## Q.2 What is the difference between parameters and arguments?

Parameter are just like placeholders(variables) defined in the function declaration, while arguments are the actual values passed when calling the function.

## Q.3 What is a Function Declaration?

A function declaration is a function defined using the `function` keyword directly. It is fully hoisted and can be called before its declaration.

## Q.4 What is a Function Expression?

A function expression is a function stored inside a variable. Unlike function declarations, it is not fully hoisted.

## Q.5 What is an Arrow Function?

An arrow function is a shorter syntax of declaring functions introduced in ES6. And unlike function declarations, we can declare them only inside variables.

## Q.7 Difference between Function Declaration and Expression?

Function declarations are fully hoisted, whereas function expressions are not. Function declarations can be called before definition but function expression cannot.

## Q.8 What is Hoisting?

Hoisting is JavaScript's behavior of moving declarations to the top of their scope before execution.

## Q.9 What are First-Class Functions?

Functions are called first-class citizens because they can be assigned to variables, passed as arguments, and returned form other functions.

## Q.10 What are Higher-Order functions?

A Higher-Order function is a function that either takes one or more functions as arguments or returns a function as its result.
