# Fundamentals of JavaScript

## Q.1 What is JavaScript?

JavaScript is a High level, Object-Oriented, Multi-Paradigm programming language.

1. **Programming Language** - A type of language to instruct computers to do something.

2. **High Level** - We don't have to worry about complex things like managing the computer's memory while it runs a program because JavaScript does it automatically.

3. **Object-Oriented** - It means that it mostly based on objects for storing most kinds of data.

4. **Multi-Paradigm** - It means that it is so flexible and versatile that we can use all kinds of different programming styles such as imperative and declarative programming. And these different styles are just different ways of structuring our code.

### And

JavaScript is a high-level, interpreted, dynamically typed, single-threaded programming language primarily used to make web pages interactive.

* Single-Threaded Programming - A program executes tasks sequentially, doing only one thing at a time.

It was originally created to run inside browsers but today it can run on:

- **Browsers**
- **Servers**
- **Mobile apps**
- **Desktop apps**
- **IoT devices**

## Q.2 What is the role of JavaScript in Web Development?

JavaScript allows developers to add dynamic and interactive effects to any webpage. We also use it to manipulate the **HTML** or the **CSS**, load data from remote servers, build entire applications in the browser which we call **Web Applications**.

## Q.3 Why was JavaScript created?

JavaScript was created in **1995** by **Brendan Eich** at **Netscape**.

It was developed to add interactivity to web pages because HTML and CSS alone could not handle dynamic behavior.

## Q.4 Is JavaScript a Programming language or Scripting language?

JavaScript is both Programming and Scripting language. Historically it was called a scripting language because it was only used inside browsers.

Today it is considered a full-fledged programming language because it supports :-

- **OOP**
- **Functional Programming**
- **Modules**
- **Server-Side Development**
- **Application Development**

## Q.5 How does JavaScript run in browser?

1. Browser loads webpage
2. JS engine reads code
3. Engine converts code
4. Executes instructions
5. Updates Webpage

## Q.6 What is ECMAScript?

ECMAScript is the official specification that defines that how JavaScript should work. JavaScript follows ECMAScript standards.

## Q.7 Is JavaScript Compiled or Interpreted?

Modern JavaScript is both.

JS engines use:

- **Parsing**
- **Compilation**
- **Optimization**
- **Execution**

**Example:**
Chrome's V8 compiles code before running it for better performance.

## Q.8 Why is JavaScript Called Multi-Paradigm?

Because it supports:

**Procedural Programming** - Step-by-step execution.

**Object-Oriented Programming** - Objects and inheritance.

**Functional Programming** - Functions as values.

## Q.9 What are the Main Components of JavaScript Runtime?

**JavaScript Engine** - Executes code.

**Call Stack** - Tracks execution.

**Web APIs** - Provided by browser.

**Callback Queue** - Stores completed tasks.

**Event Loop** - Coordinates everything.
